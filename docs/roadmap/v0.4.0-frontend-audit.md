## Bug #1 — Sidebar "Rejoindre" → homepage

### Problème

**Fichier :** `src/app/presentation/shared/sidebar/auth-sidebar.component.html`

```html
<!-- Ligne ~66 — routerLink="/" est FAUX -->
<a routerLink="/" class="auth-sidebar__sublink" (click)="onNavigate()">
  <span>Rejoindre</span>
</a>
```

L'entrée "Rejoindre" dans le groupe "Sessions" du sidebar mène à `/` (landing page). Il n'existe **aucune route dédiée** `/lobby/join`. Le formulaire de rejoindre est embarqué dans `FrontPanelComponent` sur la landing. Conséquence : si l'utilisateur est déjà sur `/`, le clic n'a aucun effet visible.

### Spec de correction

**Option A (recommandée) — Créer une page `/lobby/join` dédiée :**

1. **Nouveau composant** : `src/app/presentation/features/lobby/join/join-session.component.ts`
   - Même structure que `CreateSessionComponent` (max-width 640px, card dashed)
   - Formulaire : un champ `InputComponent` pour le code de session (label : "Code de la partie")
   - Bouton `ButtonComponent` variant `primary` : "Rejoindre"
   - Logique : appelle `JoinSessionUseCase` (déjà existant) puis navigue vers `/play/{shareCode}`
   - Flèche retour standard (voir Bug #4)

2. **Route** : dans `lobby.routes.ts`, ajouter :
   ```typescript
   { path: 'join', component: JoinSessionComponent, canActivate: [authGuard] },
   ```

3. **Sidebar** : changer `routerLink="/"` → `routerLink="/lobby/join"` dans `auth-sidebar.component.html`

4. **Landing** : Le `FrontPanelComponent` côté `side='join'` peut rester pour les visiteurs non-authentifiés, mais pour les authentifiés on peut afficher un lien "Aller à la page rejoindre →" plutôt que dupliquer la logique.

## Bug #2 — Lettres pré-chargées vs lettres tapées (police différente)

### Problème

**Fichier :** `src/app/presentation/shared/grid/inputs/letter/letter-input.component.scss`

Dans le composant de jeu, trois états visuels coexistent :

| État | Classe CSS | Couleur | Graisse | Source |
|------|-----------|---------|---------|--------|
| Pré-chargé (au join) | *(aucune)* | `var(--color-ink)` = #131313 | 400 (normal) | Absence d'auteur |
| Mes lettres tapées | `.letter--mine` | `var(--color-forest)` = #255736 | **700** (bold) | `letterAuthors` = 'me' |
| Lettres d'un autre | `.letter--other` | `#8B6914` (brun) | **600** | `letterAuthors` = userId |

Le service `LetterAuthorService` ne trace les auteurs **que pour les lettres posées pendant la session WebSocket en cours**. Les lettres venant de `fullDto.cells` à l'initialisation (et après chaque `resync()`) n'ont pas d'auteur → elles s'affichent toujours en style par défaut.

### Spec de correction

**Fichier :** `src/app/presentation/features/grid/play/grid-player.component.ts`

Lors du chargement initial (`fullDto.cells`) et du resync, enregistrer les auteurs connus :

```typescript
// Dans initFromFullDto() ou la méthode qui peuple this.cells depuis fullDto.cells
for (const cell of fullDto.cells) {
  if (cell.type === CellType.LETTER && cell.letter) {
    const authorId = cell.letter.lastModifiedBy; // champ à vérifier dans le DTO
    if (authorId) {
      const label = authorId === this.currentUserId ? 'me' : authorId;
      this.letterAuthorService.setAuthor(cell.x, cell.y, label);
    }
  }
}
```

Si le DTO ne transporte pas `lastModifiedBy`, deux alternatives :
- **Alternative A** : marquer toutes les lettres pré-chargées comme `'preloaded'` et créer une classe `.letter--preloaded` avec un style neutre mais cohérent (ex. même graisse 600, couleur `var(--color-ink)` légèrement atténuée `opacity: 0.75`).
- **Alternative B** (UX recommandée) : Traiter les lettres pré-chargées comme "validées" et les afficher avec une couleur neutre distincte, ex. `#5a6057` (couleur des cases noires, plus douce que l'encre brute). Cela signale que la lettre existait avant la session.

**CSS à ajouter dans `letter-input.component.scss`** :

```scss
.letter--preloaded {
  color: var(--color-ink);
  font-weight: 500;        // Intermédiaire entre 400 (défaut) et 700 (mine)
  opacity: 0.8;
}
```

## Bug #3 — Code de session dans le panneau latéral (trop encombrant)

### Problème

**Fichier :** `src/app/presentation/features/grid/play/grid-player.component.scss`

Le bloc `.play-code-card` (code de partage) est dans `.play-side` (panneau latéral 17rem). Il occupe ~80px de hauteur verticale dans ce panneau étroit. La classe CSS `.play-header__code` existe dans `play-header.component.scss` (lignes 72-81) mais **n'est pas utilisée dans le template HTML** du header.

### Spec de correction

**Déplacer le code de session dans le `PlayHeaderComponent`** :

1. **`play-header.component.html`** — Ajouter le code de partage dans la zone du header :
   ```html
   <div class="play-header__code" title="Code de la partie — cliquer pour copier"
        (click)="copyCode()" [class.copied]="codeCopied">
     <span class="play-header__code-value">{{ shareCode }}</span>
     <svg class="play-header__code-icon" ...><!-- icône copier --></svg>
   </div>
   ```

2. **`play-header.component.ts`** — Ajouter `@Input() shareCode: string` et logique de copie :
   ```typescript
   @Input() shareCode = '';
   codeCopied = false;
   copyCode() {
     navigator.clipboard.writeText(this.shareCode);
     this.codeCopied = true;
     setTimeout(() => this.codeCopied = false, 1500);
   }
   ```

3. **`play-header.component.scss`** — Activer et affiner le style `.play-header__code` existant :
   ```scss
   .play-header__code {
     display: flex;
     align-items: center;
     gap: 6px;
     padding: 4px 10px;
     border: 1px dashed var(--color-forest);
     border-radius: var(--radius-sm);
     cursor: pointer;
     font-family: var(--font-label);
     font-size: var(--text-sm);
     font-weight: 700;
     letter-spacing: 0.12em;
     color: var(--color-forest);
     transition: background 0.15s;
     &:hover { background: var(--color-forest-wash); }
     &.copied { background: var(--color-forest-wash); }
   }
   ```

4. **`grid-player.component.html`** — Passer `shareCode` au `PlayHeaderComponent` :
   ```html
   <cocro-play-header [shareCode]="sessionShareCode()" .../>
   ```

5. **`grid-player.component.html` + `.scss`** — Supprimer le bloc `.play-code-card` du panneau latéral.

## Bug #4 — Flèche retour : présente sur certaines pages, absente sur d'autres

### Problème

**Incohérence de navigation :**

| Composant | Route | Sidebar visible ? | Flèche retour ? |
|-----------|-------|-------------------|-----------------|
| `CreateSessionComponent` | `/lobby/create` | ✅ Oui | ✅ `← CoCro` |
| `MySessionsComponent` | `/lobby/mine` | ✅ Oui | ❌ Non |
| `MyGridsComponent` | `/grid/mine` | ✅ Oui | ❌ Non |
| `GridEditorComponent` | `/grid/create` et `/grid/:id/edit` | ✅ Oui | ❌ Non |
| `LoginComponent` | `/auth/login` | ❌ Non | ✅ `← CoCro` + "Revenir à l'accueil" |
| `RegisterComponent` | `/auth/register` | ❌ Non | ⚠️ `← CoCro` seulement (pas de "Revenir à l'accueil") |
| `JoinSessionComponent` | `/lobby/join` (à créer) | ✅ Oui | À définir |

### Règle UX à appliquer

> **Règle :** Les pages avec le sidebar n'ont **pas besoin** de flèche retour (le sidebar assure la navigation). La flèche retour n'est nécessaire que pour les pages **sans sidebar** (pages d'auth).

### Spec de correction

1. **`CreateSessionComponent`** — **Supprimer** la flèche `← CoCro` (redondant avec le sidebar).
   La page a le sidebar → pas besoin de navigation manuelle.

2. **`RegisterComponent`** — Aligner avec `LoginComponent` :
   Ajouter un lien "Revenir à l'accueil" en bas du formulaire (comme sur la page login).

3. **Pages avec sidebar** (`MyGridsComponent`, `MySessionsComponent`, `GridEditorComponent`) — Ne pas ajouter de flèche retour. La navigation via sidebar est suffisante.

4. **`GridPlayerComponent`** — Le bouton "Quitter" est correct (il quitte aussi la session WebSocket, c'est intentionnel). Pas de changement.

## Bug #6 — Incohérences typographiques entre composants

### Problème

Les composants n'utilisent pas systématiquement les variables CSS de tokens pour les polices.

**Polices définies dans `_tokens.scss`** (à vérifier) :
- `--font-title` : `'Caveat', cursive` (titres décoratifs)
- `--font-hand` : `'Patrick Hand', cursive` (UI informel)
- `--font-body` : `'Crimson Text', serif` (corps de texte)
- `--font-label` : `'Spline Sans', sans-serif` (labels UI)
- `--font-grid` : `'Comic Neue', cursive` (lettres dans la grille)

**Usages hardcodés trouvés à corriger :**

| Fichier | Ligne | Code hardcodé | Correction |
|---------|-------|---------------|-----------|
| `my-grids.component.scss` | h1 | `font-family: 'Caveat', cursive` | → `font-family: var(--font-title)` |
| `my-grids.component.scss` | état vide | `font-family: 'Patrick Hand', cursive` | → `font-family: var(--font-hand)` |
| `grid-card.component.scss` | titre grille | `font-family: 'Patrick Hand', cursive` | → `font-family: var(--font-hand)` |
| `grid-card.component.scss` | tooltip nom | `font-family: 'Patrick Hand', cursive` | → `font-family: var(--font-hand)` |
| `my-sessions.component.scss` | couleur danger | `var(--color-danger, #c0392b)` | → `var(--color-red)` |

**Taille de titre incohérente entre pages :**

| Composant | Taille h1 | Police h1 |
|-----------|-----------|-----------|
| `MyGridsComponent` | 2.6rem hardcodé | `'Caveat'` hardcodé |
| `MySessionsComponent` | `var(--text-2xl)` = 1.5rem | héritage |
| `CreateSessionComponent` | héritage | héritage |

**Spec :** Standardiser tous les titres de page (`h1`) en :
```scss
h1.page-title {
  font-family: var(--font-title);
  font-size: clamp(1.8rem, 3vw, 2.4rem);
  color: var(--color-forest);
  font-weight: 400;
}
```
Créer cette classe dans `styles.scss` (global) pour réutilisation.

## Bug #7 — Incohérences de layout (max-width, radius, border)

### Problème

**Max-widths des pages :**

| Page | Max-width |
|------|-----------|
| `LandingHomeShellComponent` | 1100px |
| `MyGridsComponent` | 1080px |
| `MySessionsComponent` | 900px |
| `CreateSessionComponent` | 640px |
| `GridEditorComponent` | (pleine largeur) |

L'absence de standardisation crée une impression de désalignement visuel lors de la navigation entre pages.

**Border-radius des cards :**

| Composant | Variable | Valeur |
|-----------|---------|--------|
| `CreateSessionComponent` | `--radius-lg` | 18px |
| `MySessionsComponent` | `--radius-panel` | 22px |
| Play side cards | `--radius-md` | 14px |

**Border style des cards :**

| Composant | Border |
|-----------|--------|
| `CreateSessionComponent` | `var(--border-dashed)` (pointillés) |
| `MySessionsComponent` | `var(--border-dashed)` (pointillés) |
| Play side cards | `var(--border-soft)` (trait continu) |

### Spec de correction

Définir dans `_tokens.scss` :
```scss
// Layout standards
--page-max-width-narrow: 680px;   // Formulaires seuls (create, join)
--page-max-width-mid: 920px;      // Listes de cards (sessions, grilles)
--page-max-width-wide: 1100px;    // Dashboards et pages riches

// Card standard
--card-radius: var(--radius-lg);  // 18px partout
--card-border: var(--border-dashed); // dashes partout dans les listes
```

Appliquer ces variables dans tous les composants concernés.

## Bug #9 — Variable `--color-danger` non définie

### Problème

**Fichier :** `src/app/presentation/features/lobby/my-sessions/my-sessions.component.scss`

Usage : `color: var(--color-danger, #c0392b)` — la variable `--color-danger` n'existe pas dans `_tokens.scss`. La valeur de fallback `#c0392b` est proche mais légèrement différente de `--color-red` (#B83225).

### Spec de correction

**Option A** : Remplacer par `var(--color-red)` dans `my-sessions.component.scss`.

**Option B** : Ajouter dans `_tokens.scss` :
```scss
--color-danger: var(--color-red); // Alias sémantique
```

Recommandation : Option B pour permettre des usages sémantiques futurs (danger ≠ rouge pur dans certains contextes).
