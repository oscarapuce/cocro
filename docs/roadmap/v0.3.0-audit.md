## 2. Angular DDD Architecture Audit

### 2.1 Architecture Rules (from CLAUDE.md)

```
domain/         → models, rules, services (pure functions, no Angular deps)
application/    → services, ports (interfaces), use cases, DTOs
infrastructure/ → adapters (HTTP, localStorage, STOMP), auth, guards, interceptors
presentation/   → features (pages), shared components
```

**Dependency rules:**
- `domain/` → NOTHING (pure TypeScript, no imports from other layers)
- `application/` → `domain/` only (ports are interfaces + InjectionTokens)
- `infrastructure/` → `application/` + `domain/` (implements ports)
- `presentation/` → `application/` + `domain/` (NOT `infrastructure/` directly)

### 2.6 Violation Summary

| ID | Severity | Description | Fix |
|----|----------|-------------|-----|
| **V1** | 🔴 High | `SessionHttpAdapter` has no port interface — `MySessionsComponent` imports concrete class directly | Create `SessionManagementPort` + wire in `rootConfig.ts` |
| **V2-V6** | 🟡 Medium | 5 presentation components import `AuthService` directly instead of using `AUTH_PORT` | Replace `inject(AuthService)` → `inject(AUTH_PORT)` and add `token()`, `logout()`, `isAnonymous()`, `isPlayer()`, `currentUser()` to `AuthPort` |
| **V7** | 🟡 Medium | `FrontPanelComponent` imports both `AuthService` and `getNetworkErrorMessage` from infrastructure | Use `AUTH_PORT` + move error message helper |
| **V8** | 🔴 High | `MySessionsComponent` injects `SessionHttpAdapter` (concrete class) directly — bypasses hexagonal boundary | Create port + use case |
| **V9** | 🟡 Medium | `GridPlayerComponent` imports `AuthService`, `mapGridTemplateToGrid`, and `getNetworkErrorMessage` from infrastructure | Inject via port; move mapper to application; move error util to application |
| **V10-V11** | 🟠 Low | `CreateSessionComponent` and `MyGridsComponent` import `getNetworkErrorMessage` from infrastructure | Move util to `@application/` or `@domain/` |

### 2.7 Cross-cutting Issues

| # | Issue | Impact |
|---|-------|--------|
| **C1** | `getNetworkErrorMessage()` lives in `@infrastructure/http/` but is used by 4+ presentation components | Should be in `@application/` (it operates on domain-level error semantics) |
| **C2** | `mapGridTemplateToGrid()` lives in `@infrastructure/adapters/session/` but is called from `GridPlayerComponent` (presentation) | Should be in `@application/` (it maps DTO → domain model, which is an application concern) |
| **C3** | `AuthPort` is too thin — doesn't expose `token()`, `logout()`, `currentUser()`, signal-based `isAuthenticated()`, `isAnonymous()`, `isPlayer()` | Extend `AuthPort` to cover all needed presentation capabilities, or create a dedicated `AuthQueryPort` |
| **C4** | `SaveGridUseCase` uses `@Inject()` constructor pattern while all others use `inject()` | Inconsistency — migrate to `inject()` for uniformity |

## 6. Remediation Spec

### 6.1 DDD Fixes

#### 6.1.1 — Create `SessionManagementPort` (fixes V1, V8)

```
File: application/ports/session/session-management.port.ts
```

```typescript
export interface SessionManagementPort {
  getMySessions(): Observable<SessionSummary[]>;
  deleteSession(shareCode: string): Observable<void>;
}
export const SESSION_MANAGEMENT_PORT = new InjectionToken<SessionManagementPort>('SESSION_MANAGEMENT_PORT');
```

Wire `SessionHttpAdapter implements SessionManagementPort` in `rootConfig.ts`.
Create `GetMySessionsUseCase` and `DeleteSessionUseCase` in `application/use-cases/`.
Update `MySessionsComponent` to inject the port via use cases.

**Effort:** 2h

#### 6.1.2 — Extend `AuthPort` (fixes V2–V7, V9)

Add to `AuthPort`:
- `token(): string | null`
- `currentUser(): Signal<AuthResponse | null>`
- `isAuthenticated: Signal<boolean>`
- `isAnonymous: Signal<boolean>`
- `isPlayer: Signal<boolean>`
- `logout(): void`

Then replace all `inject(AuthService)` in presentation with `inject(AUTH_PORT)`.

**Effort:** 3h

#### 6.1.3 — Move `getNetworkErrorMessage` to application (fixes V10–V11, C1)

Move `getNetworkErrorMessage()` from `@infrastructure/http/network-error.ts` to a new `@application/error/error-message.util.ts`. Update all imports. Keep `NetworkError` class and `toNetworkError()` in infrastructure (they depend on `HttpErrorResponse`).

**Effort:** 1h

#### 6.1.4 — Move `mapGridTemplateToGrid` to application (fixes V9, C2)

Move `grid-template.mapper.ts` from `@infrastructure/adapters/session/` to `@application/mappers/`. It maps DTO → domain model, which is an application-layer responsibility. Update imports in `GridPlayerComponent` and adapter.

**Effort:** 30min

#### 6.1.5 — Migrate `SaveGridUseCase` to `inject()` (fixes C4)

Replace `constructor(@Inject(GRID_PORT) ..., @Inject(EDITOR_DRAFT_PORT) ...)` with `inject(GRID_PORT)` + `inject(EDITOR_DRAFT_PORT)`.

**Effort:** 15min
